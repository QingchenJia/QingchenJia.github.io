## 简介

VibeCoding Session Manager 是一款命令行工具，用于统一管理 AI 编程助手的会话与技能资源。它面向 **Claude Code**、**Codex (OpenAI)**、**GitHub Copilot** 三类代理，帮助你集中查看、检索、清理和迁移会话与技能文件，避免磁盘被无感占用，也让跨代理协作更顺畅。

一句话概括：**把分散在多个代理里的会话与技能，拉回到一个清晰、可控的管理入口。**

## 产品定位

- **工具类型**：CLI 终端工具，专注于“会话与技能”的管理
- **使用方式**：本地扫描、按需操作、交互式删除与检索
- **面向人群**：频繁使用多个 AI 编程助手的个人或团队

## 核心亮点（两点）

1. **跨代理统一管理会话与技能**
    - 会话：`list` / `delete` / `delete-id` / `prune` / `inspect` / `search`
    - 技能：`skills` / `skills register` / `skills deregister` / `skills diff`
    - 价值：不用再切换多个工具或目录，一次操作覆盖多代理
2. **可视化统计 + 健康检查**
    - `stats` 启动本地仪表盘，查看会话概览、存储占用、用量与配额
    - `doctor` 扫描异常、权限问题、孤儿文件等
    - 价值：既能看全局趋势，也能主动发现潜在问题

## 详细功能说明

### 会话管理

- **列表与筛选**：按代理分组展示会话数量、体积、最近活动
- **交互式删除**：支持多选删除与二次确认，避免误删
- **按天数清理**：`prune -d 30` 快速清理长期未使用会话
- **会话检索**：按关键词跨会话搜索，定位历史对话
- **会话详情**：查看会话摘要、路径、体积、时间等信息

### 技能管理

- **统一总览**：一张表看到哪些技能在哪些代理已注册
- **跨代理迁移**：在一个代理装好技能后，一键同步给其他代理
- **差异对比**：检查同名技能在不同代理间是否一致

### 数据统计与健康检查

- **可视化仪表盘**：本地启动服务，图表展示会话数量与存储占用
- **配额与用量**：展示使用量趋势与配额信息（与代理自身数据源保持一致）
- **异常扫描**：识别空文件、索引异常或孤儿文件等

## 适用场景

- 磁盘压力大：一键筛选并清理旧会话，快速释放空间
- 跨代理协作：把同一套技能快速同步到其他代理
- 项目复盘：检索历史对话、定位关键会话并查看明细
- 团队规范化：统一管理会话与技能，减少隐性维护成本
- 学习沉淀：搜索过往问题与解法，建立个人知识库

## 安装方式

要求：Node.js >= 18

```bash
git clone https://github.com/QingchenJia/vibecoding-session-manager.git
cd vibecoding-session-manager
npm install -g .
```

安装后即可使用：

```bash
vibe --help
```

## 快速体验

```bash
vibe list
vibe delete
vibe prune -d 30
vibe skills
vibe stats
vibe doctor
```

## 使用建议（更高效的流程）

1. 每周或每月运行一次 `vibe list` 和 `vibe prune -d 30`
2. 新增技能后，用 `vibe skills register` 同步到其他代理
3. 遇到问题先用 `vibe search` 查历史，减少重复沟通成本
4. 空间异常时用 `vibe doctor` 做一次健康检查

## 推荐理由

- **真正解决痛点**：会话与技能长期堆积的管理问题有明确解决路径
- **覆盖面广**：一套工具支持三大主流代理生态
- **上手简单**：命令清晰、交互友好、无需额外配置
- **结构化管理**：从“零散文件”提升到“可追踪资产”

如果你经常与多种 AI 编程助手协作，或在意工作区整洁与可维护性，这个工具非常值得加入日常工具箱。
